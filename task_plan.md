## 已做决策
| 决策 | 理由 |
|------|------|
| 使用项目内 Markdown 规划文件跟踪进度 | 任务跨资料整理、网站实现、部署，需可恢复状态 |
| 默认创建静态网站并使用 GitHub Pages 部署 | GitHub Pages 对静态资源和 Markdown/HTML 阅读站点最直接 |
| 默认使用 GitHub 账号 `ForeverHYX` 和仓库名 `iotsec` | 当前目录名为 `iotsec`，GitHub CLI 已登录为该账号 |
| 使用 Python 直接抽取 PPTX/PDF 文本 | 本机缺少 LibreOffice、pandoc、poppler 等转换工具 |
| GitHub 仓库创建为 public | 用户明确要求“仓库默认设为public” |
| 静态站点不使用构建依赖 | GitHub Pages 可直接发布仓库根目录，降低部署复杂度 |
| 添加 `.nojekyll` | 防止 Pages 进入 Jekyll 处理流程，按原样托管静态文件 |
| 使用 GitHub Actions 部署 Pages | legacy Pages build 失败且无详细错误；workflow artifact 部署已成功 |
| 历年卷以“回忆版 + 模拟补全”形式呈现 | 用户提供的是考后回忆碎片，需要补足题干、选项与参考答案；必须避免误称为原卷逐字复刻 |
| 扩展站点搜索到正文索引 | 历年卷常按题目关键词检索，如 hidden、WEP、RFID，仅搜标题不足以支撑复习 |
| 章节自测答案使用原生 `<details>` 折叠块 | 题目默认可先自测，点开后看参考答案；不引入新框架，静态站点仍可直接托管 |
| 课程笔记从“速背检查”升级为“零基础讲解” | 用户明确要求假设读者什么都不懂，看完能理解 PPT 涵盖知识点 |

## 遇到的错误

| zsh 将 `source[branch]` 当 glob 处理 | 1 | 使用引号重新运行 GitHub Pages API 命令 |
| legacy GitHub Pages build failed | 1 | 添加 GitHub Actions Pages workflow 并切换 `build_type=workflow` |
