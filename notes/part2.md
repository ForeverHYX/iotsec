# 第三周 Part 2：蜂窝网络安全

## 零基础导读

LTE 安全要先理解最小架构。手机叫 UE，基站叫 eNodeB，手机和基站之间的无线链路叫空口。空口协议栈可以粗略理解为：PHY 负责真正的无线符号和子帧，MAC 负责无线资源调度，RLC 负责分段和重传，PDCP 负责加密和完整性相关处理，RRC 负责连接控制、测量、切换和安全模式配置。LTE 一帧是 10 ms，里面分子帧；PCFICH 告诉 UE 控制区域长度，PDCCH 承载下行控制信息和调度，PDSCH 承载实际下行数据。PSS/SSS 帮 UE 做小区搜索和时域同步，pilot signal/reference signal 帮 UE 估计信道，GPSDO 可给攻击设备提供高精度频率/时间参考。很多广播/控制信息必须让未认证或刚接入的 UE 能读，所以“有加密”不等于所有控制信息都有真实性保护。

SigOver 利用的是物理层 capture effect：如果两个信号在同一资源上到达，接收机可能解码更强或更同步的那个。攻击者不是解密用户数据，而是在特定子帧发恶意信号，覆盖合法基站的控制/广播内容。为了让 UE 接受，攻击者要听合法信号，构造匹配的 malicious subframe，还要靠 PSS/SSS、pilot signal、GPSDO 等方法做时域和频域同步。它和普通 jamming 不同：jamming 是让你听不清，SigOver 是让你听清攻击者想让你听的内容。

ReVoLTE 则是密码实现细节问题。LTE 用户面常用流密码，密钥流由 `Key/COUNT/Bearer/Direction/Length/EEA` 等输入决定。只要这些输入组合唯一，密钥流就不会重复；如果 RRC Security Mode Command 后连续 VoLTE 通话出现 COUNT 重置且 Bearer ID 重用，就可能重用同一密钥流。攻击者录下第一通密文，再打第二通让受害者说可预测内容，就能恢复第二通密钥流，再解第一通对应片段。

## 本章知识地图

1. **LTE 最小背景**：UE、eNodeB、RRC/PDCP/RLC/MAC/PHY、frame/subframe、OFDM symbol、控制信道和数据信道。
2. **SigOver 攻击链**：capture effect -> 监听合法配置 -> 构造 malicious subframe -> PCFICH/PDCCH/PDSCH 等资源上覆盖 -> UE 解码攻击者内容。
3. **信令风暴**：伪造控制信息诱导大量 UE 做 location update 或服务请求，核心网信令压力被放大。
4. **ReVoLTE 密码链条**：IMS/RTP 语音 -> LTE Bearer -> 流密码输入 -> COUNT/Bearer 重复 -> keystream 重用 -> XOR 泄露。
5. **工程教训**：规范中的 nonce、Bearer、Direction、Length、COUNT 这些字段不是细枝末节，唯一性错误会破坏强密码算法。

## 初学者常见疑问

问：SigOver、FBS/伪基站、jamming 有什么区别？

答：jamming 主要让合法信号无法被解码，目标是破坏可用性。FBS/伪基站通常诱导 UE 接入攻击者基站，UE 的连接对象变了。SigOver 中 UE 仍连接合法 eNodeB，但攻击者在某些子帧用更强、更同步的信号覆盖合法内容，让 UE 在局部资源上解码攻击者的 PCFICH/PDCCH/PDSCH 等信息。

问：为什么广播控制信息不能都加密？

答：手机刚开机、刚入网或正在找小区时，还没有和网络建立完整安全上下文。如果所有系统信息都加密，新 UE 就不知道如何接入。于是很多广播/控制信息必须公开可读，这就带来真实性问题：别人看得懂不一定危险，但别人能伪造并让 UE 接受就危险。

问：Bearer 和 COUNT 为什么会影响 ReVoLTE？

答：Bearer 是核心网到 UE 的逻辑承载，可以理解成一条带标识的业务通路；COUNT 是流密码输入中的计数器/nonce 部分。流密码安全依赖“同一 key 下每个包的输入唯一”。如果新的语音承载没有正确换 Bearer ID，COUNT 又重置，就可能生成和上一通相同的 keystream，导致 `C1 XOR C2 = P1 XOR P2`。

## 1. 本章速览

本章围绕 LTE 安全中的两类现实攻击：信号遮蔽攻击 SigOver 和 VoLTE 密钥流重用攻击 ReVoLTE。考试重点是理解蜂窝网络不是“有加密就安全”：广播控制信息、物理层捕获效应、计数器/承载标识等实现细节都可能成为攻击面。

## 2. LTE 基础

- LTE 是 3GPP 定义的 4G 蜂窝技术，从 GSM/UMTS 演进而来。
- LTE 完全分组交换，目标是更高数据率和更低时延。
- 空口协议栈包括 RRC、PDCP、RLC、MAC、PHY 等层。
- LTE 帧结构严格按时间组织，典型一帧 10 ms，包含多个子帧。
- UE 和 eNodeB 之间的控制与数据都依赖精确同步和无线资源配置。

## 3. SigOver 信号遮蔽攻击

- **基本原理**：攻击者利用信号捕捉效应，发射比合法基站更强或更精确覆盖的恶意信号，使 UE 在特定子帧上解码攻击者内容。
- **攻击者不需要密钥**：攻击目标通常是广播/控制信息；攻击者不能解密受害者与基站之间的加密流量，但可以监听合法广播信息并构造恶意子帧。
- **关键条件**：
  - 识别受害者接入基站的物理配置。
  - 构造与合法通信配置匹配的恶意信息。
  - 在时域和频域上与合法信号足够同步。
- **与伪基站区别**：伪基站诱导 UE 接入攻击者基站；SigOver 是在受害者仍连接合法基站时，局部覆盖某些下行信号。
- **安全含义**：SigOver 利用的是物理层捕获效应和广播控制面的真实性不足，不是破解 LTE 加密。答题时要强调它能影响控制路径和可用性，却不等于能直接读出用户加密数据。

## 4. SigOver 的应用与防御

- **信令风暴**：攻击者可诱导大量 UE 产生位置更新或服务请求，不需要控制大量恶意设备也能放大核心网信令压力。
- **放大效率数字**：PPT 中的 **45 vs 21600** 不是设备数量，而是每个 UE 每小时的请求率对比：正常 location-update/service requests 约 45 次/小时，SigOver 诱导下可到约 21600 次/小时；**675 vs 432000** 是正常 all signaling requests 与 SigOver 诱导后的对比，量级约 640 倍。这些数字的意义不是背精确模型，而是说明控制面诱导可把单个 UE 的信令请求频率放大成大规模信令风暴。
- **检测思路**：利用合法基站信道和攻击信道的差异进行检测，但非视距 NLOS、移动性和环境变化会降低鲁棒性。
- **签名广播信息**：对广播信息加数字签名可提升真实性，但全球漫游需要公钥体系，证书管理和广播低周期带来开销与兼容挑战。

## 5. VoLTE 与 ReVoLTE

- VoLTE 使用 LTE 分组网络承载语音，UE 与 IMS 建立联系，语音数据通过 RTP 等协议承载。
- LTE 用户面通常使用流加密。流加密的安全前提是同一密钥流不能重复使用。
- **ReVoLTE 攻击核心**：在 UE 与基站保持连接、连续两次通话时，如果 `COUNT` 被重置且 `Bearer ID` 没有正确变化，可能导致两次 VoLTE 流量使用相同密钥流。
- **攻击流程直觉**：
  1. 攻击者监听受害者第一次加密通话。
  2. 攻击者随后主动与受害者建立第二次通话，并诱导产生已知明文语音。
  3. 如果密钥流重用，已知明文可恢复第二次密钥流，再用于解出第一次通话对应片段。
- **防御**：不同通话使用不同 Bearer ID，避免 COUNT/密钥流重用；运营商侧检测配置错误；确保用户面加密参数唯一。
- **实验和防御补充**：ReVoLTE 论文实验使用 3 台 UE 和 USRP B210 等设备验证攻击可行，报告可恢复约 89% 的目标数据包，主要损失来自未捕获到的下行 packets。防御除了确保 Bearer ID/COUNT 唯一，还可以在应用层加密 RTP 流量，使 LTE 层密钥流重复也难以直接得到语音明文。
- **公式直觉**：如果两段 RTP 语音分别为 `P1`、`P2`，却使用同一密钥流 `K`，攻击者得到 `C1=P1 XOR K` 和 `C2=P2 XOR K`。当第二通话有已知明文时，可恢复 `K` 并解出第一通话对应片段。
- **实现教训**：流密码安全不仅依赖算法，还依赖 nonce、计数器、承载 ID、方向位等输入组合的唯一性。任何重复都会把强算法降级成可分析的 XOR 关系。

## 6. 考试重点

- 能解释 LTE 中广播控制信息为什么是攻击面。
- 能说明 SigOver 与伪基站/FBS 的差异。
- 能解释捕获效应、时域同步、频域同步在 SigOver 中的作用。
- 能描述 SigOver 如何导致信令风暴。
- 能说明流加密为什么绝不能重用同一密钥流。
- 能描述 ReVoLTE 的前提：连续 VoLTE 通话、COUNT 重置、Bearer ID 不变或不正确变化。
- 能给出 SigOver 和 ReVoLTE 的防御思路。

## 7. 易混点

- **加密不保护所有广播信息**：很多系统信息必须让未认证 UE 读取，因此需要额外真实性保护。
- **不能解密不等于不能攻击**：SigOver 不解密用户数据，但可以操纵控制路径。
- **密钥重用和密码算法强弱不同**：即使算法本身安全，nonce/COUNT/Bearer 使用错误也会破坏安全。

5. ReVoLTE 中 `COUNT` 和 `Bearer ID` 分别扮演什么角色？

1. 它主要覆盖或伪造广播/控制信息，利用物理层捕获效应和控制信息真实性不足；目标不是解密用户面密文，所以不需要 UE 的会话密钥。

5. `COUNT` 是流加密 nonce/计数器的一部分，保证每个包密钥流新鲜；`Bearer ID` 区分承载，若它不变且 COUNT 重置，就可能重复同一密钥流。
