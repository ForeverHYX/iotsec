# Week 1: 课程导论、无线网络概览与安全基础

## 零基础导读

如果你从来没有学过安全，先把“安全”理解成一套保护资产的工程方法。资产可以是数据、设备、账号、无线频谱、车辆控制器，也可以是系统的可用服务。分析任何系统时，顺序不要从算法开始，而是从“我要保护什么”开始：先找资产/系统资源，再写安全目标，再找威胁、漏洞和风险，最后才选择 countermeasure，也就是安全机制。比如智能门锁的资产不是只有密码，还包括开锁记录、蓝牙配对关系、固件、云端账户和门本身的可用性。

无线和 IoT 会在本课反复出现，是因为它们把传统网络安全问题放大了。无线信号在空气里广播，攻击者不需要插网线就能监听、干扰或伪造；频谱又是稀缺资源，同一频段要被很多设备频谱复用，复用越激进，干扰管理越重要。Shannon limit 告诉我们，在给定带宽和信噪比时，信道容量有上限，所以无线系统必须在功率、带宽、编码、调制、复用和可靠性之间取舍。安全设计也必须承认这些物理限制：加密不能阻止 jamming，认证也不能让一个没电的传感器继续工作。

IoT 的核心不是“设备能上网”这么简单，而是物理世界通过传感器、通信网络、云/边缘服务和应用闭环连接起来。IoT 四层可以先记成：感知识别层负责采数据，网络层负责传数据，管理服务层负责存储和处理，应用层负责做业务决策。智能网联汽车就是综合案例：LIDAR、摄像头、GPS 和车内 CAN 总线采集状态，V2X/蜂窝/Wi-Fi 让车与车、路、云通信，云端或车端算法给出控制建议，执行器控制刹车、转向和 e-cockpit。此时网络安全会直接变成物理安全问题。

## 本章知识地图

1. **安全分析基本链条**：asset/system resource -> security objective -> threat/vulnerability/risk -> policy -> mechanism -> prevention/detection/recovery。
2. **安全目标**：CIA 是底座，认证、授权、访问控制、不可否认、问责和隐私是实际系统中常一起出现的扩展目标。
3. **无线背景**：licensed/unlicensed band、ISM、频谱复用、handoff、SINR、Shannon limit、SD radio/SDWN、Ad-hoc、能量受限无线都会影响后续协议设计。
4. **IoT 背景**：IoT = 物理对象 + sensing and recognition + M2M/网络 + fog/cloud 管理服务 + 应用控制。
5. **综合案例**：智能网联汽车 ICV/V2X 把 CPS、传感器、无线网络、云服务、控制器和执行器连在一起，是理解 IoT 风险的好例子。

## 初学者常见疑问

问：安全目标、威胁、漏洞、风险有什么区别？

答：安全目标是你希望系统保持的性质，例如机密性、完整性、可用性。威胁是可能造成损害的事件或攻击者行为，例如窃听、篡改、jamming。漏洞是系统中能被利用的弱点，例如默认口令、短 IV、缺少认证。风险是威胁利用漏洞后造成损失的可能性和影响。考试答题时最好把这四个词串起来，而不是只说“系统有安全问题”。

问：为什么无线安全不能只靠加密？

答：加密主要保护机密性，能让监听者看不懂内容，但无线还有可用性和真实性问题。攻击者可以发强干扰让合法设备收不到信号，可以伪造管理帧让设备断开，可以中继信号让距离判断失效。无线系统还受频谱复用、功率、移动性和 Shannon limit 限制，所以安全机制必须和物理层、MAC 层、网络层一起考虑。

问：IoT 四层模型为什么重要？

答：因为 IoT 攻击常常跨层发生。感知识别层的传感器可能被欺骗，网络层可能被窃听或阻塞，管理服务层可能泄露数据库，应用层可能有越权逻辑。只保护其中一层不够。例如摄像头视频传输加密了，但云账号弱密码仍会泄露画面；云端权限做对了，但摄像头被物理遮挡或假数据输入，系统仍会做错判断。

## 1. 本章速览

本章先交代课程范围，再建立后续所有章节都会用到的安全语言。课程主题覆盖无线通信基础、Wi-Fi/蜂窝安全、RFID/NFC/Bluetooth、IoT 安全、定位与侧信道等。考试上最重要的是两条主线：一是能用安全目标、威胁、漏洞、风险、策略、机制这些概念描述系统；二是能解释无线网络为什么比有线网络更难保护。

## 2. 必须掌握的概念

- **CIA 三元组**：机密性 Confidentiality、完整性 Integrity、可用性 Availability。机密性还包括隐私控制，完整性既包括数据完整性，也包括系统按预期运行。
- **身份真实性 Authenticity**：实体认证确认“你是谁”，数据认证确认“数据来自可信来源”。
- **访问控制 Access Control**：授权主体才能按权限访问资源。
- **不可否认性 Non-repudiation**：通信或操作之后不能否认。
- **问责 Accountability**：系统能把行为追踪到对应实体。
- **攻击者类型**：被动攻击者只监听；主动攻击者会篡改、重放、注入、破坏；内部攻击者在边界内，外部攻击者在边界外。
- **安全策略与机制**：策略定义“什么是安全”，机制负责强制执行。策略可以冲突，机制也可能不匹配需求。
- **预防、检测、恢复**：预防阻止违规；检测发现违规；恢复评估损害、修复系统并继续运行。
- **威胁、漏洞、风险**：威胁是可能造成损害的行为或事件，漏洞是系统中可被利用的弱点，风险是威胁利用漏洞后造成损失的可能性和影响。答题时不要只写“有攻击”，要说明攻击利用了什么弱点、破坏了哪个安全目标。
- **安全保证链条**：一个机制通常只能覆盖一部分目标。例如加密保护机密性，但不自动提供来源认证；访问控制能限制操作权限，但如果认证或日志缺失，仍可能被冒用或难以追责。

## 3. 安全设计原则

- **机制经济性**：机制越简单越容易分析和实现正确。
- **开放设计**：安全不能依赖算法或设计隐藏，应该依赖密钥等受控秘密。
- **模块化与分层**：把复杂系统拆成边界清晰的组件。
- **默认失败安全**：默认拒绝，而不是默认允许。
- **最小权限与权限分离**：实体只获得完成任务所需的最小权限，关键操作需要多个条件或角色配合。
- **最少公共机制**：减少多个用户共享同一安全敏感机制。
- **心理可接受性/最小惊讶**：安全机制要让用户容易正确使用。
- **隔离与封装**：把故障、攻击面和敏感数据限制在边界内。

## 4. 无线网络概览

- **无线系统类型**：蜂窝网络、Wi-Fi、Bluetooth、ZigBee/IEEE 802.15.4、卫星系统、无线传感器网络、Ad hoc/mesh、认知无线电、能量受限网络等。
- **蜂窝网络核心思想**：区域划分为小区，频率/时隙/码/空间资源复用；小区越小容量越高，但切换、干扰协调和控制复杂度也更高。
- **Wi-Fi 标准演进**：802.11b/a/g/n/ac/ax/be 对应不同频段、调制方式、MIMO/OFDMA/MLO 等能力。高密部署下争用、干扰和静态参数会降低体验。
- **mmWave 和 Massive MIMO**：高频段带宽大、天线小，适合大规模天线；挑战在路径损耗、信道估计和系统复杂度。
- **Bluetooth**：短距离、低成本、2.4 GHz 个人区域网络。
- **ZigBee/IEEE 802.15.4**：低速、低功耗、低成本，适合传感器和控制设备。
- **无线传感器网络 WSN**：节点多、低速率、能源受限，数据通常汇聚到中心节点，网络智能比单个设备更重要。
- **认知无线电**：在不破坏授权用户的前提下利用空闲频谱，典型范式包括 underlay、interweave、overlay。

### 无线和 IoT 入门术语小表

- **licensed/unlicensed band**：授权频段由运营商或机构独占使用，免授权频段如 ISM 可被 Wi-Fi、Bluetooth、ZigBee 等共享。共享带来便利，也带来干扰和安全边界模糊。
- **频谱复用与 Shannon limit**：频谱有限，同一频率要在不同小区、空间、时间或码上复用；Shannon limit 提醒我们容量受带宽和 SINR 约束，不能无限提高速率。
- **handoff**：移动终端从一个基站/接入点切到另一个基站/接入点。切换需要控制信令，配置错误会影响可用性和安全状态连续性。
- **SD radio 与 SDWN**：Software Defined radio 用软件定义无线收发功能；Software Defined Wireless Networking 把无线网络控制逻辑集中或可编程化，便于优化但也引入控制面安全问题。
- **Ad-hoc、分布式控制与能量受限无线**：没有固定基础设施时，节点自己协商路由和信道；电池或 energy harvesting 设备还要把通信、安全计算和寿命一起考虑。

### IoT 历史、四层与智能网联汽车

IoT 历史可以从 RFID、WSN、M2M、CPS 和普适计算几条线理解。RFID 解决“物体身份识别”，WSN 解决“物理环境采集”，M2M 解决“机器之间自动通信”，CPS 强调计算过程和物理过程闭环耦合。后来 cloud/fog、移动互联网、IPv6、AIoT 把这些能力组合成更大的 IoT 系统。

IoT 四层在 Week 1 中可先按职责记：感知识别层负责 RFID、传感器、摄像头、LIDAR、麦克风等输入；网络层负责蜂窝、Wi-Fi、LPWAN、V2X 等传输；管理服务层负责数据库、DAS/NAS/SAN 存储、IaaS/PaaS/SaaS 云服务、数据分析和设备管理；应用层负责智能交通、车联网、医疗、家居、工业等业务。管理服务层的数据问题很关键：数据来自不同设备，格式、可信度、隐私等级不同，若没有访问控制、审计和脱敏，会把感知层数据变成大规模隐私风险。

智能网联汽车 ICV/V2X 是 IoT/CPS 综合案例。车辆用 LIDAR、摄像头、雷达、GPS 感知环境，用 CAN 总线连接 ECU、刹车、转向和动力控制，用蜂窝或 V2X 与路侧、云端、其他车辆通信，用 e-cockpit 呈现给驾驶员。攻击摄像头、CAN 消息、V2X 消息、云账号或 OTA 更新，都可能从网络安全问题升级为物理安全事故。

## 5. 为什么无线安全更难

- **广播介质天然可被监听**：不需要插线或物理接入，范围内设备都可能接收信号。
- **注入和重放更容易**：攻击者可以伪造无线帧、重放旧消息、发起中间人攻击。
- **干扰与阻塞是物理层问题**：jamming 可以直接破坏可用性。
- **移动性带来动态拓扑**：位置、信道质量、邻居关系和接入点都不断变化。
- **资源受限**：许多 IoT/传感设备计算、存储、电池有限，不能直接套用复杂安全协议。
- **身份和位置更难绑定**：无线设备的 MAC 地址、信号强度、位置和用户身份之间并不天然等价，攻击者可以伪造地址、中继信号或改变发射功率。
- **答题套路**：看到无线/IoT 场景，先判断资产是什么，再从广播监听、伪造注入、重放、中继、干扰、弱口令/弱密钥、物理接触这些角度列威胁，最后对应到 CIA、认证、授权、隐私和可用性。

## 6. 考试重点

- 能完整写出 CIA，并给出每一项在无线/IoT 场景中的例子。
- 能区分真实性、访问控制、不可否认性、问责这几个容易混淆的目标。
- 能说明“策略”和“机制”的关系：策略定义目标，机制实现目标，机制必须和需求匹配。
- 能解释主动/被动、内部/外部攻击者的区别。
- 能列举安全设计原则，并说明它们如何降低系统风险。
- 能说明无线网络的共享频谱、广播传播、移动性和能量约束如何扩大攻击面。

## 7. 易混点

- **认证不等于授权**：认证解决身份，授权解决能做什么。
- **完整性不等于机密性**：加密主要保护机密性，消息认证码/签名保护完整性与来源认证。
- **开放设计不等于公开密钥**：开放的是设计，秘密仍然是密钥。
- **安全不是单一算法问题**：同一系统要同时考虑用户、策略、协议、实现、运维和成本。

| SD radio | Software-defined radio | 软件定义无线电，用软件配置调制、频率和协议，是无线实验和攻击的重要工具。 |
| SDWN | Software-defined wireless networking | 软件定义无线网络，用集中控制或可编程方式管理无线资源和策略。 |
| CPS | Cyber-Physical System | 信息系统和物理过程耦合，攻击会影响真实世界，如车辆、工厂、电网。 |
| CAN | Controller Area Network | 车内控制总线，连接 ECU；若缺少认证，伪造消息可能影响车辆控制。 |
| e-cockpit | 电子座舱 | 智能车中仪表、娱乐、导航、人机交互等集中系统，既有隐私风险也有横向移动风险。 |

安全分析固定模板：

1. 先列 Asset：数据、设备、网络、控制权限、物理安全。
2. 再列 Threat：窃听、篡改、伪造、重放、DoS、恶意节点、物理接触。
3. 找 Vulnerability：弱口令、无认证、默认密钥、未加密、协议设计缺陷。
4. 判断 Risk：攻击成功概率和后果。
5. 选择 Countermeasure：加密、认证、访问控制、审计、隔离、更新、备份和恢复。

PPT 细节补充：

- `NIST Computer Security Handbook` 对安全的经典定义强调保护系统资源，目标包括 Confidentiality、Integrity、Availability。考试中若问 security objectives，先写 CIA，再按题目补认证、审计、不可抵赖。
- `System Resource` 指系统里需要保护的任何资源：数据、程序、设备、服务、通信链路、密钥、账户和物理控制能力。
- `Adversary` 是对手/攻击者，可以是外部黑客、恶意内部人员、被攻陷设备、恶意基站或恶意标签。
- `Non-repudiability` 是不可否认性，意思是参与者事后不能否认自己发过某条消息或做过某次操作，通常依赖数字签名、日志和时间戳。
- `C = B log2(1 + SNR)` 是无线容量上限的核心直觉：`B` 是带宽，`SNR` 是线性信噪比。`spectral efficiency = data rate / bandwidth`，单位常写 bit/s/Hz，表示每 Hz 频谱能承载多少数据率。
- `SDN/NFV/SDWN`：SDN 是 Software Defined Networking，把控制平面和转发平面分离；NFV 是 Network Function Virtualization，把防火墙、网关等网络功能虚拟化；SDWN 是 Software Defined Wireless Networking，用可编程控制管理无线资源。
- `D2D` 是 Device-to-Device，终端之间直接通信，可降低基站负载和时延，但会带来发现、认证、干扰控制和隐私问题。
- `Massive MIMO` 使用大量天线做波束赋形，提高容量和覆盖；`OFDMA` 把 OFDM 子载波分给不同用户，是 Wi-Fi 6/蜂窝调度的关键；`MLO` 是 Multi-Link Operation，Wi-Fi 7 可让设备在多个链路上并行或切换以降低时延、提高吞吐。
- `Underlay / Interweave / Overlay` 是认知无线电共享频谱的三种思路：Underlay 在很低干扰约束下同时用；Interweave 发现主用户空闲时用；Overlay 利用编码/协作知识在不伤害主用户的情况下用。
- `FCC` 是美国 Federal Communications Commission，管理美国频谱和通信规则；`OSM` 可理解为 Open Spectrum Management/开放频谱管理思想；`ITU-R` 是 International Telecommunication Union Radiocommunication Sector，负责国际无线电通信建议和频谱协调。
- IoT 相关缩写：`EPC` 可指 Electronic Product Code，RFID/Auto-ID 中给物体唯一编码；`M2M` 是 Machine-to-Machine；`CPS` 是 Cyber-Physical System；`The Fog` 是靠近边缘设备的计算层，介于云和终端之间；`NB-IoT` 是窄带蜂窝 IoT；`IEEE 1588` 是精确时间同步协议。

## 历年卷风格练习

1. 写出三个基本 Security Objectives，并各举一个 IoT 例子。
2. 解释 Threat、Vulnerability、Risk 的区别，并用“智能门锁 Wi-Fi/蓝牙连接”举例。
3. 为什么智能网联汽车 ICV 属于 CPS？CAN 和 e-cockpit 分别带来什么安全风险？
4. licensed/unlicensed band 对无线安全有什么影响？

<details class="self-test-answer">
<summary>参考答案</summary>

1. Confidentiality 防止未授权读取，例如摄像头视频不能被陌生人看；Integrity 防止未授权篡改，例如门锁开关状态不能被伪造；Availability 保证服务可用，例如报警器不能被 DoS 关闭。
2. Threat 是攻击来源或事件，如附近攻击者；Vulnerability 是弱点，如蓝牙默认 PIN 或 Wi-Fi 弱密码；Risk 是两者结合后的损失，如攻击者利用弱密码开锁或获取居住习惯。
3. ICV 把计算、网络和物理驾驶控制耦合，网络攻击可能影响真实车辆动作，因此是 CPS。CAN 风险是车内控制消息缺少强认证，e-cockpit 风险是复杂应用、联网服务和用户数据可能成为入侵入口。
4. 授权频段干扰较可控但接入门槛高；免授权频段如 ISM 允许 Wi-Fi/Bluetooth 等共享，部署方便但容易拥塞、被干扰、被嗅探，也更依赖协议自身安全。
